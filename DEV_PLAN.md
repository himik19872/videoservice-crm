# 🗺️ План разработки VideoService CRM
# План разработки CRM для домофонной компании

## Технологический стек
- **Backend**: Python + Django + Django REST Framework
- **Frontend**: React + TypeScript + Ant Design
- **Mobile**: React Native
- **Database**: PostgreSQL
- **Deployment**: Docker + Nginx + Gunicorn

## Основные модули
Интеграция с Ростелеком АТС (виртуальная АТС).
Сервис для получения истории звонков (CDR) через API Ростелеком.
### 1. Управление заявками
- Создание заявок (ремонт, подключение, продажа)
- Назначение мастерам по регионам
- Смена статусов (новая → в работе → выполнена → отменена)
- История изменений

### 2. Справочник мастеров
- Список мастеров с распределением по регионам
- Контактная информация
- Статус занятости

### 3. База клиентов
- Личные данные клиентов
- История заявок
- Адреса обслуживания
- эспорт из экселя 
- синхронизация с ЕРЦ ( единый расчетный центр )

### 4. Учет оборудования
- Список оборудования (домофоны, камеры, вызывные панели)
- Серийные номера
- Гарантийные сроки

- История ремонта

### 5. Отчеты
- Количество заявок за период
- Среднее время выполнения
- Рейтинг мастеров
- Финансовая отчетность

### 6. Интеграция с телефонией и внутриний чат сотрудников
- Автоматические звонки клиентам
- TTS сообщения
- Логирование звонков
- внутриний чат сотрудников 

## Роли пользователей
- **Администратор** - полный доступ
- **Диспетчер** - создание заявок, управление клиентами
- **Мастер** - просмотр своих заявок, обновление статусов

### Интеграция с меседжером макс 
- уведомление клиентов через меседжер об статусах заявки
- уведомление мастеров о новых заявках исли они не онлайн 
- уведомление диспечеров если мастер покинул geo зону в рабочее время 

#### интеграция с внешними crm 
- интеграция с 1с
- интеграция с битрикс24


### учет времени работы и производительность сотрудников 
- учет времени работы мастеров и диспетчеров
- учет матерьялов (оборудывания )
- учет кол-во выполненых заявок мастерами 
- учет заявок обработаных диспетчеров 
- рейтинг сотрудников по количесве заявок и обработаных платных заявок 


### заявки 
- учет начилных и без наличных платежей от клиентов 
- заявки монтаж для монтажников с фото видео фиксацией качество работ 
- заявки для сервисных инженеров для проведение ТО раз в квартал или год 
Создание заявок, назначение мастеру, статусы и история по каждой.
Фильтры по статусу, мастеру и срокам - видно, что в работе и что просрочено.
Заметки к заявкам и к абонентам - важное не потеряется.



Несколько компаний (юрлиц)
Абоненты закрепляются за разными компаниями со своими реквизитами - они подставляются в квитанции.
Роли и доступы
Руководитель, бухгалтер, менеджер, техник - каждый видит только то, что положено.
Сводка при входе
Дашборд с главными цифрами: деньги, долги, просроченные заявки и график.
Каждое действие фиксируется датой и сотрудником - всегда видно, кто что сделал.
Удаление с восстановлением
Удаленные записи можно вернуть - защита от случайных ошибок.
Расчет зарплаты мастеров
Автоматический расчет оплаты труда мастеров по выполненным работам.
Выдача оборудования мастерам под отчет.
Контроль того, что у кого на руках - материальную ответственность видно в любой момент.
Каталог и движение
Каталог оборудования, приход и инвентаризация.
Полная история движений каждой единицы - от прихода до установки и списания.
